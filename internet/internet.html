Clients and Servers

Computers connected to the Web are called clients and servers.

  • Clients are the typlical Web user's Inernet-connected devices (for example, your cmputer cinnected to uour Wi-Fi, or your phone connected to your mobile network) and Web-accessing software available on those devices (usually a web broswer like Firefox or Chrome).
  • Severs are computers that store webpages, sites, orapps. When a client device wants to access a webpage, a copy of the webpage is doenloaded from the server onto the clients machine to be dispayed in the user's broswer.

So What Actually Happens?

  1. The browser goes to the DNS server and finds the real ddress of the server that the website lives on.
  2. The browser sends an HTTP request message to the server asking it to send a copy of the website to the client.
  3. If the serer approves the slient's request, the server sends the client a "200 OK" message, which means "Permission granted! Here you go!", and then starts sending the website's files to the browser as a series of small chunks.
  4. The browser assenbles the small chunks into a complete website and displays ot to you!
